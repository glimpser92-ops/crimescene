# 오르빗 돔 실종 사건

브라우저에서 혼자 플레이하는 한국어 크라임씬 조사 게임입니다. 플레이어는 천문관 전시홀에서 사라진 운석 사건을 조사하며, 현장 단서 수집, 증거 노트, 용의자 심문, 추리 보드, 최종 고발 순서로 사건을 재구성합니다.

## 실행

```bash
npm install
npm start
```

기본 주소는 `http://localhost:5173`입니다. 다른 포트가 필요하면 `PORT=8080 npm start`처럼 실행할 수 있습니다.

## 게임 흐름

1. 사건 파일을 열고 공개 브리핑을 확인합니다.
2. 브리핑에서 모든 용의자의 시간대 알리바이와 공개 관계를 확인합니다.
3. 현장 화면에서 열린 조사 지점을 선택해 단서를 확보합니다.
4. 단서를 획득하거나 추리 보드로 새 단서를 열면 동선/타임라인 패널에 관련 내용이 표시됩니다.
5. 증거 노트에서 확보한 단서를 검색하고 태그로 걸러 봅니다.
6. 심문 화면에서 용의자의 진술, 갈등 관계, 증거 제시 반응을 대조합니다.
7. 추리 보드에서 시간, 수단, 동선, 동기 범주로 증거를 연결합니다.
8. 조건을 충족하면 최종 고발장이 열리고, 정답이 맞으면 리빌을 확인합니다.

## 주요 기능

- 증거 노트는 단서 종류에 맞는 시각 모듈과 대조 포인트를 보여 줍니다.
- 동선/타임라인 패널은 확보한 단서 내용을 시각적으로 배치해 어느 시간과 이동 경로에서 모순이 생기는지 보여 줍니다.
- 심문 화면은 용의자별 관련 증거, 갈등 관계, 증거 제시 반응, 검증 상태를 요약합니다.
- 추리 보드는 증거를 시간, 수단, 동선, 동기/해소 범주로 묶어 줍니다.
- 단서 해금 힌트는 정확한 카드 조합을 알려 주지 않고, 필요한 추리 방향만 암시합니다.
- 추리 보드 연결이 성공하면 새로 열린 단서나 조사 지점을 명확히 안내합니다.
- 잘못된 최종 고발은 정답 조합을 노출하지 않고 부족한 범주만 알려 줍니다.

## 스포일러 보호와 힌트

- 첫 화면과 브리핑은 공개 정보만 보여 줍니다.
- 잠긴 단서는 숨겨진 제목, 본문, 진범 정보, 최종 단서를 노출하지 않습니다.
- 잠긴 단서의 안내는 "특정 추리 보드 연결이 필요하다"처럼 넌지시 말하고, 어떤 단서들을 연결해야 하는지는 직접 말하지 않습니다.
- 이미 획득한 단서는 노트, 심문, 동선/타임라인 패널에서 다시 확인할 수 있습니다.

## 검증

```bash
npm run test:smoke
```

스모크 테스트는 브라우저에서 전체 플레이 흐름을 확인하고 `artifacts/`에 데스크톱 및 모바일 스크린샷과 리포트를 저장합니다.

최근 검증 산출물:

- `artifacts/visual-clue-flow-report.json` - 동선/타임라인 패널, 획득 단서 표시, 비스포일러 문구 검사
- `artifacts/visual-clue-flow-desktop.png`
- `artifacts/visual-clue-flow-mobile.png`
- `artifacts/smoke-investigation-desktop.png`
- `artifacts/smoke-investigation-mobile.png`

최신 스모크 기준은 모든 추리 규칙 8개 해결, 단서 24개 도달, 최종 점수 100점입니다.

## 파일 구조

- `server.js` - 정적 파일 서버와 상태 확인 엔드포인트
- `public/index.html` - 게임 진입 HTML
- `public/app.js` - 화면 렌더링, 상태, 추리 로직, 테스트 훅
- `public/case-data.js` - 사건 정보, 증거, 용의자, 정답 데이터
- `public/styles.css` - 전체 게임 UI 스타일
- `data/scripts/smoke-test.js` - 브라우저 기반 회귀 테스트
