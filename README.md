# 과학실의 소동 — 멀티플레이어 크라임씬 웹 게임

진행자 1명 + 용의자 4명 + 형사 최대 16명이 방 코드로 접속해 플레이하는
실시간 추리 롤플레이 게임입니다. (Node.js + Express + Socket.IO)

- **사건**: "'슈퍼 로봇' 파손 및 설계도 도난 사건" — 용의자 4명, 단서 24개, 범인은 나범인
- **규칙**: 범인만 거짓말 가능. 나머지 용의자는 진실만 말함(부끄러운 사실은 질문받기 전까지 침묵 가능). 토론은 앱 밖에서.

## 실행 방법

```bash
cd web
npm install
npm start          # http://localhost:3000
```

다른 포트: `PORT=8080 npm start` 또는 `node server.js 8080`

## 화면 구성 (3종)

| 화면 | 주소 | 용도 |
|---|---|---|
| 진행자 콘솔 | `/host.html` | 방 생성, 역할 배정, 단계·타이머·투표·진상 공개 제어, 모든 비밀 정보 열람 |
| 공유 디스플레이 | `/display.html?room=코드` | 프로젝터/TV용 16:9 네온 슬라이드 (진행자 콘솔의 "공유 디스플레이 창 열기") |
| 참가자 화면 | `/` 또는 `/?room=코드` | 스마트폰용. 역할 카드, 비공개 단서·힌트, 사건 자료, 투표 |

## 진행 순서 (50~60분)

1. **대기실** — 공유 화면을 띄우고 참가자들이 방 코드로 입장 (최소 5명: 용의자 4 + 형사 1)
2. **역할 배정 → 게임 시작** — 진행자 콘솔에서 클릭. 형사 인원에 맞춰 단서 24개 자동 분배
3. **사건 브리핑** — 슬라이드 9장 (표지 → 개요 → 피해자 → 용의자 4명 → 타임라인 → 규칙)
4. **역할 확인** — 각자 휴대폰에서 비공개 역할 카드 확인
5. **단서 공유 토론 (10분)** — 형사들이 단서·힌트를 말로 공유. 진행자가 구역을 골라 공유 화면에 현장 사진+단서를 띄울 수 있음
6. **용의자 심문 (15분)** — 스포트라이트 기능으로 한 명씩 집중 심문
7. **최종 토론 (5분) → 투표** — 전원 휴대폰으로 범인 지목
8. **진상 공개** — 투표 결과 → 범인 공개 → 결정적 단서 3개 → 범행 재구성 → 다른 용의자들의 진실 → 엔딩

## 난이도 설계 (왜 너무 쉽지도 어렵지도 않은가)

- **결정적 단서 3개** (No.05 젖은 스케치 / No.19 윤활유 천 / No.20 아크릴 파편)는
  항상 **서로 다른 형사**에게 분배되어 토론 없이는 조합 불가
- 기준 단서 No.01(깨진 로봇 날개)과 No.20(신발 밑창 파편)도 다른 형사에게 분배 → 반드시 대조 필요
- **레드 헤링**이 용의자 4명 전원을 향함: 발자국·농구화(박체육), 키링(오덕후), 분노 쪽지(박체육), 소음 일지(이모범), 파란 물감(혼동 유도)
- **보조 힌트**는 정답이 아니라 "무엇과 비교할지"만 알려주는 질문형
- 진행자 콘솔의 "전체 정보 열람"에서 형사별 단서 분배와 단서 유형(결정적/레드 헤링/알리바이)을 보고 토론을 조율 가능

## 온라인 배포 (교실 밖 접속)

상태를 메모리에 저장하는 단일 Node 서버라 어디든 쉽게 배포됩니다.

- **Render** (무료): New → Web Service → 저장소 연결 → Build `npm install`, Start `npm start`
- **Railway / Fly.io**: 동일하게 `npm start`만 지정
- 같은 와이파이 안에서는 배포 없이 `http://<내PC IP>:3000` 으로 바로 접속 가능

주의: 방 상태는 서버 메모리에만 있으므로 서버를 재시작하면 진행 중인 방이 사라집니다. (방은 6시간 후 자동 정리)
